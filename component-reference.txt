================================================================================
COMPONENT REFERENCE - Go Artisan (Astro + MDX)
================================================================================

Ground truth seluruh komponen yang boleh di-import dari "@components" di dalam
modul .mdx, plus pola penulisan code block dan diagram Mermaid. Lampirkan file ini
ke Project ChatGPT bersama ai-instructions.md supaya model menulis import, prop,
slot, code block, dan diagram PERSIS sesuai sumber (anti-halusinasi).

Aturan singkat yang paling sering salah:
- Import HANYA dari "@components": Section, Box, Steps, Step, Recap, CardGrid,
  Card, Figure, Chip, Hero, Compare, FileTree, Endpoint, Def. Tidak ada komponen
  lain. Jangan mengarang.
- KODE ditulis sebagai fenced code block (```go, ```bash, ```sql, ...), BUKAN
  komponen dan BUKAN <pre> manual. Lihat bagian "CODE BLOCK" di bawah.
- DIAGRAM alur/arsitektur/urutan/state ditulis sebagai fenced ```mermaid block.
  Lihat bagian "MERMAID". SVG tangan (lewat <Figure>) hanya untuk diagram konsep
  khusus (memori, pointer) yang tak cocok di Mermaid.
- Hero: judul lewat prop `title` (HTML, set:html). JANGAN tulis <h1> di MDX.
- Chip: HANYA di slot meta Hero; `icon` dari daftar sah (lihat Chip.astro).
- Box: 5 varian sah untuk Go (tip|analogy|warn|note|bridge); `label` boleh HTML;
  `icon` emoji kecil opsional.
- Section: `title`/`sub` boleh HTML; <h2> milik Section (sub-judul pakai <h3>/<h4>).
- Tiap blok/komponen berteks ditulis SATU baris (lihat ai-instructions.md). Isi
  fenced code block & mermaid BEBAS multi-baris (itu memang blok kode).

--------------------------------------------------------------------------------
CODE BLOCK   (fenced code block — disorot Shiki, BUKAN komponen)
--------------------------------------------------------------------------------
Tulis kode sebagai fenced code block biasa. Highlight, header nama file, label
bahasa, dan tombol "Salin" dibuat otomatis oleh transformer Shiki di astro.config.
JANGAN menulis <pre>/<code> manual dan JANGAN memakai komponen untuk kode.

- Bahasa yang umum: go, bash, sh, sql, json, yaml, dockerfile, text, toml, makefile.
- Tambahkan nama file lewat meta `title="..."` setelah nama bahasa. Nama file akan
  tampil di header code block.
- Bahasa bash/sh/shell/zsh/console otomatis tampil dengan "chrome terminal"
  (tiga titik lampu + label). Untuk sesi terminal pakai title="Terminal".
- PENTING: di DALAM fenced code block, kurawal { }, backtick struct tag, panah,
  dan karakter apa pun bersifat LITERAL. Tidak perlu escape &#123;/&#125;. Inilah
  kenapa kode Go (yang penuh { }) ditulis di fenced block, bukan di teks MDX.

CONTOH (tulis persis seperti ini di .mdx):

```go title="internal/product/model.go"
package product

type Product struct {
	ID    int64  `json:"id"`
	Name  string `json:"name"`
	Price int64  `json:"price"`
}
```

```bash title="Terminal"
go mod init github.com/kamu/skincare-backend
go run .
go test ./...
```

--------------------------------------------------------------------------------
MERMAID   (fenced ```mermaid block — render via integrasi astro-mermaid)
--------------------------------------------------------------------------------
Diagram alur, arsitektur, urutan (sequence), dan state ditulis sebagai fenced
```mermaid block. Integrasi astro-mermaid mengubahnya jadi SVG di sisi klien dan
mengikuti tema baca (light -> default, dark -> dark, sepia -> tema default).

- Jenis yang berguna: flowchart LR/TD, sequenceDiagram, stateDiagram-v2,
  erDiagram, classDiagram.
- Sintaks Mermaid penuh { } [] --> | | aman di dalam fence (literal, tak di-parse MDX).
- Caption OPSIONAL: taruh satu paragraf <p class="fig-cap"> tepat di bawah blok.
- JANGAN bungkus ```mermaid di dalam komponen; tulis sebagai blok berdiri sendiri.

CONTOH (tulis persis seperti ini di .mdx):

```mermaid
flowchart LR
  FE["Frontend React"] -->|HTTP JSON| API["Go API (chi)"]
  API --> SVC["Service"]
  SVC --> REPO["Repository (pgx)"]
  REPO --> DB[("PostgreSQL")]
```

<p class="fig-cap"><b>Gambar 1.</b> Penjelasan singkat di bawah diagram.</p>

FILE: src/components/Box.astro   (callout; 5 varian sah: tip|analogy|warn|note|bridge)
--------------------------------------------------------------------------------
interface Props { variant: string; label: string; icon?: string; }
// Render: <div class={`box ${variant}`}><div.box-tag>{icon?<span.b-icon>}{label set:html}</div><slot /></div>

Varian sah untuk Go Artisan (warna dari globals.css):
- tip     (biru)  : tips, best practice, idiom Go.
- analogy (teal)  : analogi dunia nyata.
- warn    (merah) : jebakan, footgun, kesalahan umum.
- note    (netral): catatan tambahan yang tidak mendesak.
- bridge  (ungu)  : "Jembatan" — memetakan konsep Go dari JS/React/PHP/Laravel
                    yang sudah dikenal pembaca. Inilah varian khas materi ini.
(Varian lama "exam" masih ada di CSS tetapi JANGAN dipakai untuk Go.)

PEMAKAIAN (MDX):
<Box variant="bridge" icon="🌉" label="Jembatan: dari try/catch ke error nilai"><p>Isi callout satu baris.</p></Box>
